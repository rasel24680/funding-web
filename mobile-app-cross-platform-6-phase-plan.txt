Cross-Platform Mobile App Delivery Plan (5 Phases)
Total Approved Budget: GBP 2,000

====================================================================
PROJECT COMMITMENT
====================================================================

We are fully committed to delivering this project in 5 focused phases with milestone-based approvals and payments.
Each phase includes practical execution, validation, and handover.


====================================================================
PHASE 1 - FIGMA DESIGN + SCOPE FINALIZATION
====================================================================
Duration: 5 days
Budget: GBP 450

What We Will Do
1. Create complete Figma design system for the mobile app.
2. Design key user journeys in Figma (onboarding, login, funding flow, application flow, dashboard, documents).
3. Build clickable prototype for client walkthrough.
4. Finalize feature scope from approved designs.
5. Lock implementation plan for development.

Phase Transactions
- T1.1: Figma first draft shared.
- T1.2: Design revision round completed.
- T1.3: Final Figma prototype approved.
- T1.4: Phase 1 payment released.

Budget Justification
- This phase defines all visual and interaction logic; changes later are costlier without this depth.


====================================================================
PHASE 2 - MOBILE APP FOUNDATION + AUTHENTICATION
====================================================================
Duration: 5 days
Budget: GBP 380

What We Will Do
1. Setup cross-platform project architecture (Android + iOS).
2. Implement navigation structure and reusable UI base components.
3. Build register/login/session flow.
4. Connect core API layer with error handling and loading states.
5. Validate secure access flow across both platforms.

Phase Transactions
- T2.1: Foundation build demo delivered.
- T2.2: Authentication flow approved.
- T2.3: API connectivity verification completed.
- T2.4: Phase 2 payment released.

Budget Justification
- Stable architecture and security controls are mandatory before feature-heavy development.


====================================================================
PHASE 3 - CORE USER JOURNEYS (FUNDING + APPLY)
====================================================================
Duration: 7 days
Budget: GBP 520

What We Will Do
1. Build funding form journey with smooth step-based UX.
2. Build lender results and eligibility display screens.
3. Build direct application journey from result to submission.
4. Add full field validation and submission states.
5. Add retry/error handling for weak network conditions.

Phase Transactions
- T3.1: Core flow alpha demo completed.
- T3.2: Validation pass completed.
- T3.3: End-to-end apply flow approved.
- T3.4: Phase 3 payment released.

Budget Justification
- This is the largest build phase with the highest logic density and integration workload.


====================================================================
PHASE 4 - DASHBOARD + DOCUMENTS + UX POLISH
====================================================================
Duration: 5 days
Budget: GBP 360

What We Will Do
1. Build user dashboard with application status views.
2. Build document upload and list management flows.
3. Improve transitions, state feedback, and interaction polish.
4. Ensure consistency with approved Figma design.
5. Stabilize account lifecycle actions.

Phase Transactions
- T4.1: Dashboard module demo completed.
- T4.2: Document flow demo completed.
- T4.3: UI polish sign-off completed.
- T4.4: Phase 4 payment released.

Budget Justification
- This phase closes critical post-application functionality and final user confidence layer.


====================================================================
PHASE 5 - QA + RELEASE PREP + HANDOVER
====================================================================
Duration: 4 days
Budget: GBP 290

What We Will Do
1. Cross-platform QA and bug fixing.
2. Final performance and stability checks.
3. Prepare release candidate build.
4. Final walkthrough and handover.
5. Start post-delivery support window.

Phase Transactions
- T5.1: QA completion report shared.
- T5.2: Release candidate approved.
- T5.3: Handover session completed.
- T5.4: Final payment released.

Budget Justification
- Final verification and release control reduce production incidents and support load.
